# Systembetrieb

Demos für produktionsrelevante Runtime-, Memory-, Tool-, Governance- und Observability-Bausteine.

Diese Beispiele sind bewusst kleine Architektur-Skizzen, weil viele Systembetrieb-Patterns Infrastruktur und Policies benötigen.

## Code

Eine Datei pro Systembetrieb-Pattern, zum Beispiel:

- `conversational_memory.py`
- `function_calling.py`
- `workflow_dag_durable_execution.py`
- `human_in_the_loop_approval_gate.py`
- `distributed_tracing.py`
- `integration_tests_for_agents.py`
